# 八卦炉与神之锻炉技术对比及特效移植方案

## 1. 概述

Fuxi Bagua Heaven Forging Furnace（简称八卦炉）和 ForgeOfTheAntichrist（简称神之锻炉）是GTLAdditions模组中两个复杂的多方块结构，它们在物理结构和功能上有许多相似之处，但在技术实现和渲染效果上存在显著差异。本报告将详细分析这两个结构的技术细节、渲染差异，以及如何将神之锻炉的特效移植给八卦炉。

## 2. 核心技术实现对比

### 2.1 基础架构

| 特性 | 八卦炉 | 神之锻炉 |
|------|--------|----------|
| 实现类 | `GTLAddCoilWorkableElectricParallelHatchMultipleRecipesMachine` | `ForgeOfTheAntichrist` |
| 继承关系 | 标准多配方机器 | 自定义实现，支持模块化 |
| 能量传输 | 有线 | 无线 |
| 配方逻辑 | 标准多配方逻辑 | 自定义配方逻辑，支持效率提升 |
| 并行处理 | 线圈并行 + 并行舱口 | 无限并行 |

### 2.2 配方类型

| 结构 | 支持的配方类型 |
|------|----------------|
| 八卦炉 | - STELLAR_LGNITION（恒星点火）<br>- CHAOTIC_ALCHEMY（混沌炼金术）<br>- MOLECULAR_DECONSTRUCTION（分子解构）<br>- ULTIMATE_MATERIAL_FORGE_RECIPES（终极材料锻造） |
| 神之锻炉 | - DIMENSIONALLY_TRANSCENDENT_PLASMA_FORGE_RECIPES（维度超越等离子体锻造）<br>- STELLAR_FORGE_RECIPES（恒星锻造）<br>- ULTIMATE_MATERIAL_FORGE_RECIPES（终极材料锻造） |

### 2.3 结构设计

| 特性 | 八卦炉 | 神之锻炉 |
|------|--------|----------|
| 外观方块 | DIMENSION_INJECTION_CASING | GOD_FORGE_INNER_CASING |
| 旋转支持 | 非Y轴旋转 | 非Y轴旋转 |
| 模块系统 | 无 | 支持5个模块：<br>- HELIOFUSION_EXOTICIZER<br>- HELIOFLARE_POWER_FORGE<br>- HELIOFLUIX_MELTING_CORE<br>- HELIOTHERMAL_PLASMA_FABRICATOR<br>- HELIOPHASE_LEYLINE_CRYSTALLIZER |
| 特殊功能 | 线圈温度显示 | 效率随时间增长、能量消耗减少、星星仪式 |

### 2.4 性能特性

| 特性 | 八卦炉 | 神之锻炉 |

| 运行时间影响 | 无 | 有，4小时达到最大效率 |
| 并行限制 | 受线圈和并行舱口限制 | 无限并行 |

## 3. 渲染系统对比

### 3.1 渲染实现

| 特性 | 八卦炉 | 神之锻炉 |
|------|--------|----------|
| 渲染器 | 标准 `workableCasingRenderer` | 自定义 `ForgeOfAntichristRenderer` |
| 模型文件 | 使用通用融合反应堆模型 | 自定义模型和纹理 |
| 渲染类型 | 标准方块渲染 | 复杂的3D效果，包括环形结构和星星 |
| 特殊效果 | 无 | 环形旋转、星星脉动、信标光束 |
| 渲染模式 | 无 | 4种渲染模式：NORMAL、RAINBOW、COLLAPSING、RECOVERING |
| 全局渲染 | 否 | 是，视距384格 |

### 3.2 渲染效果细节

#### 八卦炉渲染
- 使用标准的 `workableCasingRenderer`，基于 `GTCEu.id("block/multiblock/fusion_reactor")` 模型
- 没有特殊的视觉效果，主要依赖方块本身的纹理
- 显示线圈温度信息

#### 神之锻炉渲染
- 使用自定义的 `ForgeOfAntichristRenderer`，实现了复杂的视觉效果
- 渲染多个旋转的环形结构，根据机器状态动态变化
- 渲染多层星星效果，包括基础层、中间层和光晕层
- 渲染信标到星星的光束
- 支持多种渲染模式，包括彩虹色和崩溃效果
- 使用缓存机制优化渲染性能
- 设置为全局渲染器，确保在远处也能看到效果

### 3.3 环结构实现原理

**神之锻炉的环结构是通过以下方式实现的：**

1. **字符数组定义结构**：在 `RingStructure.kt` 中使用三维字符数组定义了环的形状，每个字符代表一个方块
2. **方块映射**：在 `RingStructureVertexBuffer.kt` 中定义了字符到方块的映射关系
3. **顶点缓冲区**：将字符结构转换为顶点缓冲区，用于高效渲染
4. **动态渲染**：在 `ForgeOfAntichristRenderer.kt` 中实现环的旋转动画

**环的具体组成：**

| 字符 | 对应的方块 |
|------|------------|
| 'C' | GOD_FORGE_INNER_CASING |
| 'D' | GOD_FORGE_SUPPORT_CASING |
| 'E' | SUPRACHRONAL_MAGNETIC_CONFINEMENT_CASING |
| 'F' | GOD_FORGE_ENERGY_CASING |
| 'G' | REMOTE_GRAVITON_FLOW_REGULATOR |
| 'H' | SPATIALLY_TRANSCENDENT_GRAVITATIONAL_LENS |
| 'I' | CENTRAL_GRAVITON_FLOW_REGULATOR |
| 'J' | GOD_FORGE_TRIM_CASING |
| 'K' | MEDIARY_GRAVITON_FLOW_REGULATOR |

**环的渲染过程：**
1. **构建顶点缓冲区**：遍历字符结构，获取方块模型，生成顶点数据
2. **应用变换**：根据机器方向和状态应用旋转
3. **渲染环**：绑定顶点缓冲区并渲染
4. **实现动画**：通过修改旋转角度实现动画效果

```kotlin
// FuxiBaguaHeavenForgingFurnaceRenderer.kt
class FuxiBaguaHeavenForgingFurnaceRenderer(
    baseCasing: ResourceLocation,
    workableModel: ResourceLocation,
    partEntry: BlockEntry<Block>,
    partCasing: ResourceLocation
) : PartWorkableCasingMachineRenderer(baseCasing, workableModel, partEntry, partCasing) {

    @OnlyIn(Dist.CLIENT)
    override fun render(
        blockEntity: BlockEntity,
        partialTicks: Float,
        poseStack: PoseStack,
        buffer: MultiBufferSource,
        combinedLight: Int,
        combinedOverlay: Int
    ) {
        super.render(blockEntity, partialTicks, poseStack, buffer, combinedLight, combinedOverlay)
        
        if (blockEntity is IMachineBlockEntity) {
            val machine = blockEntity.metaMachine as? GTLAddCoilWorkableElectricParallelHatchMultipleRecipesMachine ?: return
            
            if (machine.recipeLogic.isWorking) {
                val tick = RenderUtil.getSmoothTick(machine, partialTicks)
                val starPos = getRotatedRenderPosition(Direction.EAST, machine.frontFacing, 0.0, 0.0, 0.0) // 调整位置以适应八卦炉结构
                val seed = blockEntity.blockPos.asLong()
                
                renderStar(tick, poseStack, buffer, seed, starPos.x, starPos.y, starPos.z)
            }
        }
    }

    @OnlyIn(Dist.CLIENT)
    override fun onAdditionalModel(registry: Consumer<ResourceLocation>) {
        super.onAdditionalModel(registry)
        registry.accept(STAR_LAYER_0)
        registry.accept(STAR_LAYER_2)
    }

    @OnlyIn(Dist.CLIENT)
    override fun hasTESR(blockEntity: BlockEntity): Boolean = true

    @OnlyIn(Dist.CLIENT)
    override fun isGlobalRenderer(blockEntity: BlockEntity): Boolean = true

    @OnlyIn(Dist.CLIENT)
    override fun getViewDistance(): Int = 384

    @OnlyIn(Dist.CLIENT)
    private fun renderStar(
        tick: Float,
        poseStack: PoseStack,
        buffer: MultiBufferSource,
        randomSeed: Long,
        x: Double,
        y: Double,
        z: Double
    ) {
        poseStack.pushPose()
        poseStack.translate(x, y, z)

        val baseRadius = 0.3f
        val middleRadius = baseRadius + 0.01f
        val outerRadius = middleRadius * 1.02f

        val cache = getOrCreateCache(randomSeed)
        val rotationSpeedMultiplier = 1.0f

        // 使用固定的白色，移除变色功能
        val starColor = 0xFFFFFFFF

        RenderUtils.renderStarLayer(
            poseStack, buffer, STAR_LAYER_2, middleRadius,
            cache.rotation2.axis, cache.rotation1.getAngle(tick * rotationSpeedMultiplier),
            starColor, RenderType.translucent()
        )

        RenderUtils.renderStarLayer(
            poseStack, buffer, STAR_LAYER_0, baseRadius,
            cache.rotation1.axis, cache.rotation0.getAngle(tick * rotationSpeedMultiplier),
            starColor, RenderType.solid()
        )

        RenderUtils.renderHaloLayer(
            poseStack, buffer, outerRadius,
            cache.rotation0.axis, cache.rotation0.getAngle(tick * rotationSpeedMultiplier),
            HALO_TEX, STAR_LAYER_2,
            1.0f, true
        )

        poseStack.popPose()
    }

    @OnlyIn(Dist.CLIENT)
    private data class RenderCache(val seed: Long) {
        val rotation0: RotationParams
        val rotation1: RotationParams
        val rotation2: RotationParams

        init {
            val random = RandomSource.create(seed)
            rotation0 = RenderUtils.createRandomRotation(random, 2.0f, 3.0f)
            rotation1 = RenderUtils.createRandomRotation(random, 0.9f, 1.5f)
            rotation2 = RenderUtils.createRandomRotation(random, 0.9f, 1.5f)
        }
    }

    @OnlyIn(Dist.CLIENT)
    companion object {
        private val STAR_LAYER_0 = GTLAdditions.id("obj/star_layer_0")
        private val STAR_LAYER_2 = GTLAdditions.id("obj/star_layer_2")
        private val HALO_TEX = GTLAdditions.id("textures/block/obj/halo_tex2.png")
        private val CACHE_MAP = ConcurrentHashMap<Long, RenderCache>()

        private fun getOrCreateCache(seed: Long): RenderCache =
            CACHE_MAP.computeIfAbsent(seed) { RenderCache(it) }
    }
}
```

**2. 创建顶点缓冲区**

```kotlin
// BaguaRingStructureVertexBuffer.kt
@OnlyIn(Dist.CLIENT)
object BaguaRingStructureVertexBuffer {

    val ringBuffers: Array<VertexBuffer> by lazy {
        try {
            Array(3) { tier ->
                buildRingBuffer(BaguaRingStructure.RINGS[tier])
            }
        } catch (e: Exception) {
            GTLAdditions.LOGGER.error("Failed to build bagua ring VertexBuffers", e)
            Array(3) { VertexBuffer(VertexBuffer.Usage.STATIC) }
        }
    }

    val BLOCK_MAPPER: Map<Char, Block> by lazy {
        mapOf(
            'A' to GTLAddBlocks.DIMENSION_INJECTION_CASING.get(),
            'B' to GTLAddBlocks.GRAVITON_FIELD_CONSTRAINT_CASING.get(),
            'C' to GTLAddBlocks.ULTIMATE_STELLAR_CONTAINMENT_CASING.get()
            // 更多映射...
        )
    }

    private fun buildRingBuffer(structure: Array<Array<String>>): VertexBuffer {
        val buffer = VertexBuffer(VertexBuffer.Usage.STATIC)
        val bufferBuilder = Tesselator.getInstance().builder
        bufferBuilder.begin(VertexFormat.Mode.QUADS, DefaultVertexFormat.BLOCK)

        val poseStack = PoseStack()
        val random = RandomSource.create()

        val centerX = structure.size / 2.0
        val centerY = structure[0].size / 2.0
        val centerZ = structure[0][0].length / 2.0

        for (x in structure.indices) {
            val plane = structure[x]
            for (y in plane.indices) {
                val row = plane[y]
                for (z in row.indices) {
                    val letter = row[z]
                    if (letter == ' ') continue

                    val block = BLOCK_MAPPER[letter] ?: continue
                    if (block == Blocks.AIR) continue

                    val visibleFaces = getVisibleFaces(structure, x, y, z)
                    if (visibleFaces.isEmpty()) continue

                    val blockState = block.defaultBlockState()
                    val model = ClientUtil.blockRenderer().getBlockModel(blockState)

                    poseStack.pushPose()
                    poseStack.translate(
                        (x - centerX),
                        (centerY - y - 1),
                        (z - centerZ)
                    )

                    val light = LightTexture.pack(
                        block.getLightEmission(blockState, EmptyBlockGetter.INSTANCE, BlockPos.ZERO),
                        13
                    )

                    renderBlockModelFaces(
                        model, blockState, visibleFaces,
                        light, poseStack, bufferBuilder, random
                    )

                    poseStack.popPose()
                }
            }
        }

        buffer.bind()
        buffer.upload(bufferBuilder.end())
        VertexBuffer.unbind()

        return buffer
    }

    // 其他辅助方法...
}
```

```kotlin
// 在 FuxiBaguaHeavenForgingFurnaceRenderer.kt 中添加环形渲染
@OnlyIn(Dist.CLIENT)
private fun renderAllRings(
    machine: GTLAddCoilWorkableElectricParallelHatchMultipleRecipesMachine,
    poseStack: PoseStack,
    partialTicks: Float,
    isWorking: Boolean
) {
    val tick = if (isWorking) RenderUtil.getSmoothTick(machine, partialTicks) else 0f
    val ringPos = getRotatedRenderPosition(Direction.EAST, machine.frontFacing, 0.0, 0.0, 0.0)

    RenderSystem.enableBlend()
    RenderSystem.enableDepthTest()
    RenderSystem.blendFunc(
        GlStateManager.SourceFactor.SRC_ALPHA,
        GlStateManager.DestFactor.ONE_MINUS_SRC_ALPHA
    )
    RenderSystem.setShader { GameRenderer.getRendertypeSolidShader() }
    RenderSystem.setShaderTexture(0, InventoryMenu.BLOCK_ATLAS)
    Minecraft.getInstance().gameRenderer.lightTexture().turnOnLightLayer()

    BaguaRingStructureVertexBuffer.ringBuffers.forEachIndexed { index, buffer ->
        poseStack.pushPose()
        poseStack.translate(ringPos.x, ringPos.y, ringPos.z)

        when (machine.frontFacing) {
            Direction.NORTH -> poseStack.mulPose(Axis.YP.rotationDegrees(270f))
            Direction.SOUTH -> poseStack.mulPose(Axis.YP.rotationDegrees(90f))
            Direction.WEST -> poseStack.mulPose(Axis.YP.rotationDegrees(0f))
            Direction.EAST -> poseStack.mulPose(Axis.YP.rotationDegrees(180f))
            else -> {}
        }

        if (isWorking) {
            val direction = if (index % 2 == 0) 1.0f else -1.0f
            val speedMultiplier = 0.4f + (index * 0.4f)
            val angleOffset = index * 120f
            val rotationAngle = (tick * speedMultiplier * 2.0f * direction + angleOffset) % 360.0f

            poseStack.mulPose(Quaternionf().fromAxisAngleDeg(1.0f, 0.0f, 0.0f, rotationAngle))
        }

        buffer.bind()
        buffer.drawWithShader(
            poseStack.last().pose(),
            RenderSystem.getProjectionMatrix(),
            RenderSystem.getShader()!!
        )
        VertexBuffer.unbind()
        poseStack.popPose()
    }

    RenderSystem.disableDepthTest()
    RenderSystem.disableBlend()
}
```

## 5. 技术建议

### 5.1 性能优化建议

1. **渲染性能**
   - 使用顶点缓冲区（VertexBuffer）存储环形结构，减少渲染调用
   - 实现视距检查，只在距离足够近时渲染复杂效果
   - 使用缓存机制存储计算结果，避免重复计算

2. **内存优化**
   - 使用对象池管理渲染相关对象，减少垃圾回收
   - 合理管理纹理和模型资源，避免内存泄漏

3. **代码结构**
   - 将渲染逻辑与游戏逻辑分离，提高代码可维护性
   - 使用接口和抽象类，便于扩展和修改

### 5.2 功能扩展建议

1. **渲染模式**
   - 实现多种渲染模式，如彩虹模式、脉动模式等
   - 添加配置选项，允许用户自定义渲染效果

2. **交互效果**
   - 添加粒子效果，增强视觉体验
   - 实现声音效果，与视觉效果同步

3. **游戏机制**
   - 考虑添加类似神之锻炉的效率增长系统
   - 实现无线能量传输功能

### 5.3 兼容性建议

1. **版本兼容性**
   - 确保代码与不同版本的Minecraft和Forge兼容
   - 使用条件渲染，在低性能设备上自动降低效果

2. **模组兼容性**
   - 确保与其他模组的渲染系统兼容
   - 避免与其他模组的渲染效果冲突

## 6. 总结

通过本报告的分析，我们可以看到八卦炉和神之锻炉在技术实现和渲染效果上的差异。将神之锻炉的特效移植给八卦炉是一个可行的目标，通过分步骤实现，可以为八卦炉添加中心光球和旋转环形等视觉效果，提升其视觉吸引力。

移植过程需要注意性能优化和代码结构，确保新的渲染效果不会影响游戏性能。同时，可以考虑添加配置选项，让用户根据自己的设备性能和个人喜好调整渲染效果。

通过这些改进，八卦炉可以在保持其原有功能的基础上，获得与神之锻炉相媲美的视觉效果，为玩家提供更加沉浸式的游戏体验。
