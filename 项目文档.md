# GTLAdditions 项目文档

## 1. 项目概述

GTLAdditions 是一个基于 GregTech Leisure (GTL) 模组包的扩展模组，旨在为其添加新的机器、配方和功能，同时保持原有的硬核科技框架。该模组通过引入大量创新设计和平衡优化，为玩家带来更流畅、更具策略性的自动化体验。

### 1.1 核心功能

- 添加多种新型多方块机器，替代原有机器并提供更高效率
- 引入新的机器部件，如无线激光舱口、超级输入双舱口等
- 添加简化的配方，优化大规模生产线
- 实现无线能量传输功能
- 集成其他模组，如 Applied Energistics 2、Jade 等

## 2. 项目结构

GTLAdditions 采用标准的 Minecraft Forge 模组结构，主要代码位于 `src/main/java/com/gtladd/gtladditions` 目录下。项目使用 Kotlin 和 Java 混合开发，利用 Gradle 作为构建系统。

### 2.1 主要目录结构

```
src/main/java/com/gtladd/gtladditions/
├── api/             # 公共 API 和接口定义
├── client/          # 客户端渲染相关代码
├── common/          # 通用功能实现
│   ├── blocks/      # 方块定义
│   ├── items/       # 物品定义
│   ├── machine/     # 机器实现
│   ├── material/    # 材料定义
│   └── recipe/      # 配方类型
├── config/          # 配置相关
├── data/            # 数据生成
├── events/          # 事件处理
├── integration/     # 与其他模组的集成
├── mixin/           # Mixin 修改
├── network/         # 网络通信
└── utils/           # 工具类
```

## 3. 核心组件

### 3.1 主类

**GTLAdditions.kt** 是模组的主入口点，负责初始化各种组件和注册事件监听器：

- 初始化创意模式标签
- 加载配置
- 初始化网络通信
- 注册事件监听器
- 注册配方类型和机器定义

### 3.2 机器系统

GTLAdditions 扩展了 GregTech 的机器系统，添加了多种新型机器和部件：

#### 3.2.1 多方块机器

- **SuperFactory mk[1,4]**：运行来自 `plants` 的配方，作为通用生产机器
- **Atomic Transmutation Core**：使用 EM Resonance Conversion Field 配方，比 Block Conversion Room 更高效
- **Lucid Etchdreamer**：使用 Photon Matrix Etch 配方，无需光刻胶即可工作
- **Astral Convergence Nexus**：使用 Space Assembler Module 配方
- **Nebula Reaper**：同时使用 Space Miner Module 和 Space Drilling Module 配方
- **Arcanic Astrograph**：使用 Cosmos Simulation 配方，作为 Eye of Harmony 的替代品
- **Arcane Cache Vault**：使用 Packer 配方，使用多配方逻辑
- **Draconic Collapse Core**：使用 Aggregation Device 配方，提供更多并行处理能力
- **Titan Crip Earthbore**：使用 Tectonic Fault Generater 配方，生产基岩粉
- **Biological Simulation Laboratoy**：使用 Biological Simulation 配方，从实体中生产资源
- **Dimensionally Transcendent Chemical Plant**：使用 Large Chemical Reactor 配方，使用多配方逻辑
- **Quantum Syphon Martix**：使用 Voidflux Reaction 配方，生产空气系列产品
- **Fuxi Bagua Heaven Forging Furnace**：使用多种配方，包括 Stellar Lgintion、Chaotic Alchemy 等
- **Antientropy Condensation Center**：使用 Antientropy Condensation 配方，无需液氦即可工作
- **Taixu Turbid Array**：使用 Chaos Weave 配方，生产废料箱、UU 放大器和 UU 物质
- **Inferno Cleft Smelting Vault**：使用 Pyrolyse Oven 和 Cracker 配方，使用多配方逻辑
- **Skeleton Shift Rift Engine**：使用 Decay Hastener 配方，更高效

#### 3.2.2 机器部件

- **Laser Hatch**：比原版更大的激光舱口，电流可达 16777216A 和 67108863A
- **Wireless Laser Hatch**：类似 Laser Hatch，但无需布线
- **Huge Output Dual Hatch**：类似于 Huge Input Dual Hatch
- **Huge Steam Input Hatch**：类似于 Large Steam Input Hatch，将蒸汽多方块配方限制提高到 HV，持续时间变为 1t
- **Super Input Dual Hatch**：类似于 Huge Input Dual Hatch，但具有更大的输入空间
- **WirelessOpticalDataHatch** 和 **WirelessOpticalComputationHatch**：允许多方块机器共享
- **AutoConfigurationMaintenanceHatch** 系列：允许多方块机器共享，并具有可获得不同奖励的插槽

### 3.3 材料系统

GTLAdditions 扩展了 GregTech 的材料系统，添加了新的材料和元素：

- 在 `GTLAddMaterial.kt` 和 `MaterialAdd.kt` 中定义
- 包括新的材料集，如 eternity、spacetime 等
- 添加了新的物品形式，如 dense cosmic plate、infinity plate 等

### 3.4 配方系统

- 添加了简化的配方，优化大规模生产线
- 引入了新的配方类型，如 Biological Simulation、Chaos Weave 等
- 添加了新的 SOC 芯片和晶圆，实现更高效的电路生产

## 4. 技术实现

### 4.1 网络通信

- 使用 Forge 的网络系统实现客户端和服务器之间的通信
- 定义了 `MachineDeltaPacket` 和 `SyncDimensionMachinesPacket` 等数据包

### 4.2 渲染系统

- 为新型机器实现了自定义渲染器，如 `ForgeOfAntichristRenderer`、`HeartOfTheUniverseRenderer` 等
- 支持不同的渲染模式和效果

### 4.3 配置系统

- 使用 `ConfigHolder` 管理模组配置
- 允许用户自定义各种参数，如超级模式缓冲区的大小

### 4.4 事件处理

- 处理各种游戏事件，如玩家维度变化、服务器生命周期等
- 实现了 `MECapabilityHandler` 处理 ME 能力

### 4.5 模组集成

- 与 Applied Energistics 2 集成，添加了 ME 超级模式缓冲区
- 与 Jade 集成，添加了自定义信息提供者
- 与 GTCEu、GTLCore、GTMThings 等模组集成

## 5. 依赖关系

GTLAdditions 依赖于多个其他模组：

| 模组名称 | 版本要求 | 用途 |
|---------|---------|------|
| GTLCore | >= 1.2.2.3 | 核心依赖 |
| GTCEu | 1.20.1-1.4.4 | GregTech 基础功能 |
| GTMThings | 1.3.5.a | 提供额外的机器和功能 |
| Applied Energistics 2 | 15.2.1 | 提供 ME 系统集成 |
| Jade | 1.20.1-forge-11.9.2 | 提供 HUD 信息 |
| ExtendedAE | 1.20-1.1.12-forge | AE2 扩展 |
| TravelAnchors | 1.20.1-5.0.1 | 提供传送点功能 |
| Stargate Journey | - | 提供星际门功能 |

## 6. 安装方法

1. 下载 GregTech Leisure 模组包
2. 删除其中的 GTLCore 并安装最新版本
3. 将 GTLAdditions 模组添加到 `/mods` 文件夹
4. 启动游戏，享受增强的 GregTech 体验

## 8. 总结

GTLAdditions 是一个功能丰富的 GregTech Leisure 扩展模组，通过添加新的机器、部件和配方，为玩家提供了更丰富、更高效的自动化体验。该模组不仅保持了 GregTech 的硬核科技风格，还引入了许多创新设计，使游戏体验更加流畅和策略性。

通过精心的代码组织和模块化设计，GTLAdditions 为 Minecraft 模组开发提供了一个很好的参考案例，展示了如何扩展现有模组并添加新功能。
